1:1 매핑 카드

아래 카드는 현재 구역 → 참고 레퍼런스 → 실제 적용법 순으로 정리한 미리보기입니다. 한눈에 “아, 이런 느낌으로 바뀌겠구나”를 보시라고 구성했습니다.

01. 첫 화면 구조를 로그인 중심에서 웨딩 히어로 중심으로

지금 첫 화면은 웨딩 소개보다 관리자 로그인 박스가 메인처럼 보입니다. 이를 브랜드 메시지 + 날짜/장소 + 메인 CTA가 먼저 보이는 구조로 바꾸는 매핑입니다.

최우선
현재 구역
  • 중앙 로그인 카드가 화면의 거의 전부
  • 웨딩 본문보다 운영자 진입점이 먼저 보임
참고 레퍼런스
Lazyweb Material 3
  • 간결한 헤더 + 큰 제목 + CTA 2개
  • 정보 우선순위가 바로 읽히는 구조
메인 카피 날짜·장소 청첩장 보기

적용 방식

첫 화면 상단에는 신랑·신부 이름, 예식 날짜, 장소, 짧은 초대 문구를 두고, 메인 버튼은 청첩장 보기 또는 오시는 길로 둡니다. 로그인은 운영자 링크로 한 단계 뒤로 보내면, 웨딩 사이트다운 첫인상이 살아납니다.

02. 버튼 위계를 관리자용 버튼이 아니라 사용자용 CTA 중심으로

현재는 로그인 버튼 하나가 가장 강합니다. 이를 메인 CTA / 보조 CTA / 관리자 링크로 역할을 나눠야 화면이 덜 딱딱하고 더 자연스럽습니다.

상
현재 구역
  • 강한 버튼이 로그인 하나뿐
  • 사용자 행동 유도보다 운영자 행동 유도가 먼저 보임
참고 레퍼런스
shadcn/ui Material 3
  • 메인 버튼 / 보조 버튼 / 아웃라인 버튼 차이 명확
  • 웨딩 톤으로 바꾸기 쉬운 둥근 버튼 구조
Primary Secondary Admin Link

적용 방식

메인 버튼은 청첩장 보기, 보조 버튼은 오시는 길 또는 축하 메시지 남기기로 두고, 관리자는 아주 작은 outline 버튼이나 텍스트 링크로 이동시킵니다. 색은 로즈베이지 또는 세이지 포인트 하나만 강하게 주는 편이 좋습니다.

03. 헤더는 짧고, 메뉴는 적게, 우측 CTA는 분명하게

현재는 첫 화면 상단 구조가 명확한 네비처럼 읽히지 않습니다. 헤더를 브랜드 / 짧은 메뉴 / 1개 CTA 구조로 잡으면 화면이 한층 현대적으로 보입니다.

상
현재 상태
  • 상단 구조보다 중앙 카드가 더 강하게 읽힘
  • 메뉴 체계가 첫 인상에서 분명하지 않음
참고 레퍼런스
Lazyweb oh-my-design
  • 메뉴 수가 적고, 우측 CTA가 또렷함
  • 내비가 콘텐츠를 방해하지 않음
청첩장 웨딩사진 FAQ 관리자

적용 방식

헤더는 로고 옆에 청첩장 · 웨딩사진 · FAQ · 관리자 정도만 두고, 우측에 오시는 길 또는 참석 의사 전달 버튼을 배치합니다. 메뉴를 줄이면 첫 화면이 훨씬 덜 복잡해 보입니다.

04. 본문 섹션은 선보다 면과 여백으로 구분

지금은 섹션들이 존재하더라도 첫인상에서는 잘 드러나지 않습니다. 배경톤 변화와 큰 여백으로 챕터감 있는 페이지로 재구성하는 매핑입니다.

상
현재 상태
  • 첫 화면 기준 섹션 전환감이 약함
  • 웨딩 본문보다 기능 화면처럼 보임
참고 레퍼런스
Material 3 oh-my-design
  • 표면과 배경을 선보다 톤 차이로 구분
  • 섹션 간 상하 여백이 충분해 리듬감이 있음
히어로 예식 정보 갤러리 FAQ

적용 방식

배경은 아이보리, 카드 표면은 화이트/크림으로 두고, 예식 정보·갤러리·FAQ·오시는 길 섹션마다 상하 여백을 크게 줍니다. 모든 구역을 같은 평면에 두지 말고, 미세한 배경 차이로 구역감이 느껴지게 합니다.

05. 사진 갤러리는 부속이 아니라 감정선을 만드는 메인 장치로

웨딩 사이트에서 사진은 단순 섹션이 아니라 전체 무드를 결정합니다. 감성 사진 중심 레이아웃으로 전환하는 매핑입니다.

중상
현재 상태
  • 첫인상에서 사진의 존재감이 거의 보이지 않음
  • 브랜드 정서보다 기능 구조가 앞섬
참고 레퍼런스
Dribbble shadcn/ui
  • 사진이 메인 무드를 만든 뒤, 정보가 따라오는 구조
  • 캐러셀/썸네일/대표 사진 배치가 자연스러움
대표 사진 썸네일 사진 더 보기

적용 방식

대표 사진 1장을 크게 두고, 아래에 3~6장의 썸네일 또는 캐러셀을 배치합니다. 갤러리는 버튼으로만 들어가는 게 아니라, 첫 화면 아래에서 바로 감정선을 이어주는 역할로 배치하는 편이 좋습니다.

06. FAQ는 아이콘 버튼이 아니라 질문형 아코디언으로

FAQ는 클릭 버튼 하나로 숨기기보다, 실제 질문이 먼저 보이는 형식이 훨씬 친절합니다. 웨딩 사이트에서 정보 신뢰감을 높이는 핵심 포인트입니다.

중
현재 상태
  • FAQ가 버튼형 진입점으로만 읽힘
  • 질문 내용이 첫눈에 보이지 않음
참고 레퍼런스
shadcn/ui Material 3
  • 질문을 리스트로 보여주고, 답만 접었다 펼 수 있음
  • 카드 안에서 FAQ를 정돈하기 쉬움
주차 안내 식 시작 시간 아이 동반 마음 전하기

적용 방식

FAQ는 주차는 어디에 하나요?, 예식은 언제 시작하나요? 같은 질문을 직접 보여주는 아코디언으로 바꾸는 편이 좋습니다. 웨딩 사이트에서는 “정보를 찾기 쉽게 한다”는 인상이 매우 중요합니다.

07. 전체 무드는 웨딩 감성, 구현 방식은 정돈된 시스템으로

무드만 예쁘고 구조가 지저분하면 오래 못 갑니다. 반대로 구조만 좋고 무드가 없으면 웨딩 사이트 같지 않습니다. 둘을 섞는 마지막 매핑입니다.

중상
현재 톤
  • 꽃 장식과 크림 배경은 좋지만 조금 정적일 수 있음
  • 템플릿 감성에서 한 단계 더 올라갈 여지가 있음
참고 레퍼런스
Dribbble oh-my-design
  • 무드를 과장하지 않고, 여백과 위계로 세련됨을 만듦
  • 화면 챕터감이 분명하면 웨딩 톤도 더 고급스럽게 느껴짐
세리프 제목 차콜 본문 로즈베이지 포인트

적용 방식

제목은 감성적인 세리프를 유지하되 본문은 읽기 좋은 산세리프로 정리합니다. 배경은 아이보리, 카드 표면은 화이트/크림, 포인트는 로즈베이지 또는 세이지 하나만 써서 전체 색 역할을 명확히 하는 편이 좋습니다.
