하객 공개 화면이 아니라 운영자 전용 콘솔 기준으로 다시 정리한 리뷰입니다. 지금 화면의 핵심은 “감성”보다 빠른 운영 판단, 반복 작업 효율, 실수 방지입니다. 다만 프로젝트 톤은 유지해야 하므로, 전체를 차갑게 바꾸기보다 현재의 웨딩 무드를 살린 채 콘솔답게 다듬는 방향으로 잡았습니다.

현재 확인한 범위 대시보드 · 사이트 관리 · 업로드 관리
공통 문제 업무 우선순위보다 감성 비중이 조금 더 앞섬
기본 방향 보수적 개선 + 기존 톤 유지 + 운영 효율 강화

현재 콘솔 구조 진단

실제 코드와 브라우저 화면을 기준으로, 각 운영 화면이 어떤 장점과 병목을 갖는지 먼저 분리했습니다.

실제 확인 기준
dashboard

운영자 홈

  • 좌측 사이드바 + KPI 카드 + 월별 차트 + 최근 생성 사이트 표 구조는 명확합니다.
  • 하지만 첫 화면에서 “무엇을 먼저 처리해야 하는지”는 약합니다.
  • 시스템 티커는 예쁘지만 정보가 길고, 경고/주의보다 정상 상태 나열에 치우쳐 있습니다.
장점: 구조 명확 문제: 우선순위 약함 문제: 알림 존재감 약함
site manage

사이트 관리

  • 검색 · 정렬 · 오늘/예정/지난 구분은 운영 흐름에 잘 맞습니다.
  • 카드 우측의 액션 버튼이 아이콘 위주라 처음 보면 역할이 즉시 안 들어옵니다.
  • 지난 결혼식 목록은 길어질수록 피로가 커질 구조입니다.
장점: 실무형 필터 문제: 액션 의미 약함 문제: 긴 목록 피로
upload manage

업로드 관리

  • 행사 선택 후 사진/하객/메시지를 관리하는 구조는 맞습니다.
  • 다만 행사 미선택 상태와 실제 데이터 0 상태가 비슷하게 보여 혼동 여지가 있습니다.
  • 실무에서는 미처리/오류/최신 업로드 같은 운영 신호가 더 앞에 오면 좋습니다.
장점: 개념 분리 문제: 빈 상태 안내 약함 문제: 운영 지표 부족

1 · 2 · 3 방안

전면 재디자인이 아니라, 지금 콘솔을 더 운영자답게 만드는 세 가지 축입니다. 1안을 먼저 하고, 2안과 3안을 순차로 얹는 방식이 가장 안전합니다.

권장 순서: 1 → 2 → 3
option 1

정보 위계 재정렬

우선
  • 대시보드 첫 화면에서 “지금 확인할 것”을 가장 위로 올립니다.
  • 정상 상태 나열보다 경고, 오늘 행사, 최근 업로드, 미완료 작업을 먼저 보이게 합니다.
  • KPI는 유지하되 설명 텍스트를 줄이고 숫자와 상태 의미를 더 또렷하게 만듭니다.
Dashboard 중심 상태 티커 압축 우선처리 카드 신설
option 2

작업 흐름 명확화

  • 사이트 관리 카드 액션을 아이콘만 두지 말고 1차 버튼/2차 버튼으로 구분합니다.
  • 행사 선택, 링크 복사, QR 확인, 하객 화면 열기 같은 반복 작업을 한 번에 이해되게 정리합니다.
  • 업로드 관리에서는 행사 미선택 상태를 더 분명히 보여주고, 선택 후 가능한 작업을 바로 제안합니다.
버튼 의미 강화 반복 작업 단축 빈 상태 가이드
option 3

톤 유지형 고급화

  • 현재의 웨딩 무드와 브라운/크림 톤은 유지합니다.
  • 대신 대비, 라벨, 경계선, 섹션 헤더를 다듬어 콘솔 특유의 정돈감을 올립니다.
  • 즉 “청첩장 서비스 느낌”은 남기되, 더 이상 랜딩처럼 보이지 않게 합니다.
기존 느낌 유지 가독성 상승 과한 리디자인 방지

실행 우선순위

실제로 손댈 때는 한 번에 다 바꾸지 말고 아래 순서로 가는 게 안전합니다.

작업 순서
1단계 — 대시보드 상단의 정보 위계를 조정해서 “오늘 할 일 / 경고 / 최근 변경”이 먼저 보이게 바꾸기
2단계 — 사이트 관리 카드 액션을 명확한 텍스트/툴팁 구조로 정리하고, 지난 결혼식 섹션 피로도 줄이기
3단계 — 업로드 관리에서 행사 미선택 상태, 빈 상태, 0건 상태를 명확히 분리하고 운영 지표 추가하기
이 파일은 운영자 콘솔 전용 리뷰입니다. 하객 화면 기준 해석은 일부러 제외했습니다. 다음 단계에서는 이 문서를 기준으로 실제 index.html / style.css 쪽 수정안으로 옮기면 됩니다.
